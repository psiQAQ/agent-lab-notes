# Zotero 指南

Zotero 是一款文献管理工具，适合用来收集、整理、标注和引用论文、网页与书籍资料。这份文档从 Zotero 本体安装开始，接着介绍浏览器扩展、中文社区、插件扩展，以及如何通过 MCP 让智能体参与文献管理。

如果你的工作流里已经有大量 PDF、笔记、标签和 collection，Zotero 的价值不只是“存文献”。真正省时间的地方在于：把导入、补全元数据、分类、去重、批量标注、生成引用这些重复操作，尽量变成可复用的流程。

Zotero 中文社区是一个开源社区，提供了较丰富的中文文档、插件整理和镜像下载入口：
<https://zotero-chinese.com/>

## 软件下载安装

<https://www.zotero.org/download/>

安装时选择默认选项即可。

## 浏览器扩展

浏览器扩展的作用很直接：把网页中的文献元数据快速保存到 Zotero，减少手动录入，方便后续整理和引用。

项目开源地址：
<https://github.com/zotero/zotero-connectors>

在 Zotero 下载页面通常会跳转到对应浏览器的插件商店页面，点击 `Install` -> `获取` -> `添加扩展` 即可。

如果没有自动跳转，也可以直接去浏览器扩展商店安装。例如 Edge 浏览器：

<https://microsoftedge.microsoft.com/addons/detail/zotero-connector/nmhdhpibnnopknkmonacoephklnflpho>

## Zotero 插件

Zotero 的很多效率提升都来自插件。常见用途包括批量下载 PDF、生成参考文献、做元数据清理，或者把若干零散操作串成一套更完整的工作流。

一般从 `插件市场` 或 `GitHub` 下载，下载后把 `xpi` 文件拖入 Zotero 的插件窗口即可完成安装。

### 1. 插件市场

Zotero 中文插件市场 / 插件商店是 Zotero 中文社区维护的子项目，不属于 Zotero 官方。它的主要价值不是“替代官方”，而是把常用插件集中整理出来，并提供更方便的国内下载入口。

插件市场地址：<https://zotero-chinese.github.io/plugins/>

建议先安装 `Add-on Market for Zotero`，后续安装其他插件会方便很多。

[插件市场 xpi 下载镜像地址](https://gh-proxy.org//https://github.com/syt2/zotero-addons/releases/download/V9.0.2/zotero-addons.xpi)

打开 `zotero`，点击 `工具` -> `插件`，将下载的 `xpi` 文件拖入窗口中即可完成安装。

回到主界面后，点击新增的 `拼图` 按钮，就可以在弹出的窗口中搜索并安装其他插件。

### 3. Zotero-Agent

#### 为什么还要自己写一个 AI 插件

目前把 AI 接到 Zotero 上的方案并不少，但大多只覆盖了某一小段流程：有的偏只读检索，有的停留在外部 MCP server 调官方 API，有的更像调试工具，有的能写但缺少权限控制、确认机制和稳定的本地工作流支持。换句话说，现成插件并不是没有，而是“不够完整”：要么不能真正深入 Zotero 进程内操作，要么写操作不够稳妥，要么工具面覆盖不到批量导入、元数据修复、去重、标签治理、工作流总结这类高频任务。

如果只是“让 AI 查一下库里有什么文献”，这些现成方案已经够用；但如果目标是让智能体长期参与本地文献管理，就需要一套更完整的能力边界，包括可控写入、默认 dry-run、权限分层、进程内扩展执行，以及围绕真实研究工作流设计的工具集合。基于这个出发点，我以 `cookjohn/zotero-mcp` 为基础，继续做成了 [Zotero-Agent](https://github.com/psiQAQ/zotero-agent)。

#### Zotero-Agent 的优势

可以把它理解成“让智能体真正能帮你整理 Zotero 文库”的插件。相较于只会查一查、读一读的方案，Zotero-Agent 更适合直接配合提示词完成一整套文献处理任务，主要优势包括：

- **支持多种方式批量导入文献**
  不仅可以扫描本地文件夹，把 PDF 批量导入 Zotero，还可以根据 DOI、arXiv ID、ISBN、PMID 这类标识符快速建条目，也支持批量导入 BibTeX、RIS、CSL-JSON 等参考文献文件。

- **能自动补全和修正文献信息**
  对新导入或信息不完整的条目，可以补全标题、作者、年份、期刊、摘要、DOI 等常见元数据，也能处理部分 DOI 修复、预印本升级到正式发表版本这类整理工作。

- **支持围绕已有文献继续扩展相关文献**
  不只是检索你库里已经有的内容，还可以根据一篇核心文献继续查它的引用、被引和相关论文，用来扩展某个研究方向的阅读列表。

- **支持检查缺失 PDF，并补下载可获取的 PDF**
  可以批量找出哪些条目缺少 PDF，并尝试补全可获取的 PDF。对于文库里“条目有了，但附件没跟上”的情况，这一点很实用。

- **适合做文库清理和批量整理**
  可以查找重复条目、合并重复项、检查元数据不完整的文献，也能批量增删改 tag、统一标签命名，并辅助整理 collection 结构。

- **适合生成综述笔记和工作流总结**
  可以读取 PDF 高亮、批注、笔记和摘要，按研究背景、方法、结论、局限性等结构生成综述草稿，也可以在项目结束时输出一份工作流总结。

- **对中文文献场景更友好**
  已经考虑到中文元数据补全和中文文献整理这类需求，比很多主要围绕英文论文场景设计的工具更实用。

- **写入前可以先预览再确认**
  很多操作可以先输出 `dry-run` 结果，也就是先看它准备怎么改，再决定是否真正写入，比较适合把智能体用于正式文库管理。

#### 安装过程

1. 安装 `zotero-agent` 插件

    在安装了前文提到的 `Add-on Market for Zotero` 插件后，`拼图` 按钮打开中文插件市场，左下角 `🔍` 搜索 `Zotero Agent`，右键点击 `安装` 即可

2. 配置智能体客户端 `MCP` 服务

   1. 方式一（推荐）：在 `zotero` -> `编辑` -> `设置` -> `Zotero Agent` -> `客户端配置`，选择正在使用的客户端，复制配置，自行配置或交给智能体代为配置。

   2. 方式二：智能体阅读配置文档指导你配置：
      - [github 版](https://github.com/psiQAQ/zotero-agent/)
      - [国内可访问 gitcode 版](https://gitcode.com/RepoPorter/zotero-agent)

#### 提示词范例

下面这些提示词比“帮我整理一下文献”更容易得到稳定结果，因为它们明确了范围、输出格式和写入前确认的要求。

* **本地 PDF 批量导入 + 元数据条目建立**
  请将本地文件夹 **xxx** 中的 PDF 批量导入 Zotero。先扫描全部 PDF，并将结果分成 **重复文件**、**已有关联条目**、**待新建条目文件** 三类。对需要新建条目的 PDF，优先提取 **DOI**、**arXiv ID**、**ISBN**、**PMID**、**标题** 等标识符，再建立标准 Zotero 条目并补全 **作者**、**年份**、**期刊/会议**、**摘要**、**DOI** 等元数据，随后加入 collection **xxx**。所有写入操作先输出 **dry-run 清单**，经我确认后再执行。

* **文献分类整理 + tag 标签治理**
  请整理 Zotero 中 collection **xxx** 的文献结构。根据 **标题**、**摘要**、**全文**、**已有 tag** 和 **注释**，先给出一份建议的子 collection 方案，并说明每个分类的判断依据。然后输出 **tag 规范化方案**，统一处理 **大小写**、**连字符**、**下划线** 和 **同义重复 tag**。在我确认前，不要实际写入，只输出 **分类表**、**tag 映射表** 和 **dry-run 计划**。

* **搜索并下载相关文献**
  请围绕研究主题 **xxx** 搜索并扩展相关文献。先检索 **Zotero 本地库**，再基于核心文献的 **引用**、**被引** 和 **相似论文** 继续扩展。对库中不存在但相关性高的文献，优先尝试通过 **DOI**、**arXiv ID**、**ISBN** 或 **PMID** 导入；如果能找到 **开放获取 PDF**，请下载并附加到对应 Zotero 条目。最后输出 **已存在文献**、**新增文献**、**无法导入文献** 和 **未能下载 PDF** 的清单。

* **文献注释综合 + 综述笔记生成**
  请读取 collection **xxx** 中所有文献的 **PDF 高亮**、**批注**、**笔记** 和 **摘要**，生成一份结构化综述笔记。请按 **研究背景**、**核心问题**、**方法路线**、**数据集**、**实验设置**、**评价指标**、**主要结论**、**局限性** 和 **可引用观点** 分类整理。每条结论都标注对应 Zotero 条目，并先展示 **综述草稿**，经我确认后再写入 Zotero note。

* **项目收尾检查 + .bib 引用生成 + 工作流总结**
  请对 Zotero 项目 **xxx** 做一次完整的文献工作流收尾检查。检查 **重复条目**、**缺失 PDF**、**缺失 DOI**、**缺失摘要**、**元数据不完整**、**tag 混乱**、**未分类文献** 和 **可能的撤稿风险**。修复前先输出 **dry-run 报告**。确认后，再基于清理后的 collection 生成 **BibTeX**，citation key 格式使用 **FirstAuthorYearShortTitle**，并额外输出一份 **工作流总结**，记录 **已完成操作**、**遗留问题** 和 **后续建议**。
